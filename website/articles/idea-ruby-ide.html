We are talking about IDE

The author has been using IDEA as a Java IDE for the past 4 years, and as Ruby IDE for the past year.
The reason that the author chooses IDEA ruby plug-in as the preferred IDE for ruby development
is based on the criteria of a good IDE, rather than that of a programming editor like VIM or SciTe. Agile software
development applies best to a project that requires good team work. This is where a good editor
is not good enough. Instead, the team needs a good “integrated development environment”, through
which developers can be as efficient as possible in their daily work of developing and coordination.

What is most important in deciding a good IDE is to know that a feature comparison list is a poor way to do it.
To quote Alan Cooper, the author of the great book “The Inmates Are Running the Asylum”.

It is abundantly clear to most of us that common folk don’t know the difference between a token ring and a mood ring. We need these creator of “faster, stronger, better” innovations to be sure that the creations actually improve our work and lives — not simply drive us crazy. We need the technology to work in the same way average people think. We need a revolution to restore our sanity.

IDEA as an superb IDE

IDEA as an IDE has surpassed its competitors due to its architecture design as well as its good
software product interaction design. Here the author will pick several big areas to show how a good
IDE can make Ruby programming easier.

File operations

You probably would think, “browse directory, open file, save file, revert file… what can be done better in this
area”. Well, that is what the author thought before he started using IDEA.

For starter, with large scale software development, the number of files has grown exponentially. With IDEA,
you can open any file by just typing the file name, and IDEA will find it in the project directory.

IDEA has this very good feature called local history. As you editing the files, IDEA records the history of them. At
the same time, IDEA tags this history based on the significant operations that you perform. If you want to find
out what the changes are since the last time you had all the tests pass, or opened the project, you can with IDEA.
And you can compare the file content, down to the lines and characters. With the local history, there is less and less
reason to keep file modified without saving it, or have a long list of file open at the same time. So IDEA became the
first product and the only IDE that finally saved the user from the idea of “save a file”, because they are saved automatically.
The old tabs can also be closed automatically so that the IDE does not get cluttered.

VCS support

With an excellent file comparison dialog, IDEA’s version control system (VCS) support is also almost perfect. The
recommended practice in a team environment is to update the source, make sure all tests pass, before check in
the changes that you have made. All these actions are at the top menus in IDEA and have default key maps.

With IDEA, you can also create and apply patches, an artifacts used most commonly in open source community. IDEA also allows you
to “Shelve” changes (temporarily remove from code and save as patch, then restore later). You can also move certain changes to
different change list, a unique feature that was only available to the customer of Perforce, one of the few successful commercial
version control system.

Plug-in Support

IDEA took a minimalist’s approach when it comes to plug-in development, which made it very easy to write or install plug-in’s
to extend its functionality even further. You specify everything in a plug.xml file. The plug-in is packaged as ONE SINGLE jar
file. The API of the plug-in framework is also well documented and very easy to use. As a matter of fact, the ruby plug-in that
JetBrain is developing right now is not the first ruby plug-in that has been built.

IDEA’s adequate Ruby support

Because of IDEA’s powerful IDE features, its Ruby support only needs to be adequate to put itself on top of the Ruby IDE list.

Ruby as a Language

IDEA understands enough about the Ruby to give you a decent syntax highlighting, error highlighting support. You can also open any
Ruby class just by typing the class name, and go to any member of a class by typing its name. If you have class specifications in rspec,
you can open them up by just typing the file name. (Snapshot).

With IDEA’s runtime support and Ruby’s script language nature, running any script is just one keystroke away (Ctrl+Shift+F10). This flexibility,
the “re-run last configuration” feature, and the ability to click on the exception stack in the output pane made it very easy to do test-driven
development with Ruby unit, or behavior-driven development with rSpec.

Explore Ruby with RDoc

IDEA has integrated RI and RDoc support, which is what most Ruby developers used to learn about different classes in Ruby. It behaves pretty
much like the fxri that came with Ruby one-click installer. You just need to type the class name and method name to see the RDoc
associated with them.

Rails Support

Even though the RoR developers that the author knows of indicated that due to the nature of Ruby, it is enough as long as you have a good IDE
support and use command line for Rails related work. Nevertheless, you can still run the generation script, add new actions and launch
the servers

Future of IDEA’s Ruby support

At the moment, IDEA is falling a bit behind on the advanced Ruby support like refactoring and code completion. IDEA was rather late in developing
its JUnit support. Hopefully this time they will be the first.

As IDEA is working on releasing Selena, the latest version with even better design and more useful features, Ruby plug-in is also under heavy
development on this platform. You can download Selena through the JetBrain EAP program, and install the Ruby plug-in through the install manager.

As an extremely satisfied Java developer with IDEA, looking at their
Development Roadmap, the author has nothing but confidence that
IDEA Ruby support will only be better and better. That could end up being a false statement, but for the moment, that is where the money is.
